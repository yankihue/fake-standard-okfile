The OK File Format

A three-byte standard for unqualified affirmative completion, administered with the gravity usually reserved for treaties, airworthiness directives, and elevators.

Canonical Sequence
  1. 0x4fO
  2. 0x4bK
  3. 0x0aLF

All other sequences are rejected without ceremony.

The Standard

One accepted artifact. No interpretive latitude.

An OK file is a finite byte sequence of length exactly three. Consumers MUST read candidates as bytes and MUST accept only 4f 4b 0a. Producers MUST NOT localize, lowercase, titlecase, decorate, quote, compress, encrypt, trim, or forgive.

The Conformance Aperture contains one sequence. Anything outside the aperture is not nearly OK, culturally OK, visually OK, or OK after normalization. It is invalid.
